# 회원관리 시스템 회원정보 연동 상세가이드

> 엑셀 기입 매크로 작업 후, 회원관리 시스템의 회원정보를 정확히 연동/업데이트하기 위한 운영 기준입니다.

---

## 적용 목적

- 개인정보 입력 항목을 표준화하여 오입력/누락을 방지
- 외국인 케이스 예외를 사전에 통제
- 저장 누락 방지를 위해 최종 업데이트 절차를 명확화

---

## 항목별 입력 기준

### 1) 생년월일

- 주민등록번호 **뒷자리까지 연동**합니다.

### 2) 주소

- `우편번호검색` 실행 후
- 주소(동) 항목에 **상세 주소까지 연동**합니다.

### 3) 직장(회사)

- 아래 형식으로 입력합니다.

```text
은행명_예금주명_계좌번호
```

예시:

```text
국민_김철수_1000-100-1002-101
```

필수 규칙:

- 구분자는 반드시 `_` 사용
- 공백(띄어쓰기) 금지
- 계좌번호는 하이픈 `-` 포함

### 4) 전화(직장)

- 동작구민인 경우 `동작구`를 입력합니다.

---

## 외국인 여부 처리(중요)

### 1) 판별 기준

- 주민등록번호 뒷자리가 **6으로 시작**하면 외국인으로 판단합니다.

### 2) 시스템 이슈

- 시스템에서 뒷자리 시작이 `6`인 주민등록번호는 에러가 발생할 수 있습니다.
- 해당 이슈는 필요 시 업체에 전달/조치 요청합니다.

### 3) 외국인 입력 규칙

- 참고사항(비고)에 아래 형식 추가 입력:

```text
외국인_뒷자리시작숫자
```

예시:

```text
외국인_6
```

- 주민등록번호 입력 시, 뒷자리 시작 숫자만 제외하고 나머지는 정상 반영합니다.

예시:

- 원본 뒷자리: `600000`
- 입력 뒷자리: `2000000`

---

## 최종 반영 절차 (가장 중요)

- 작업 완료 후 **저장**이 아니라 반드시 **수정 버튼**을 눌러 업데이트합니다.
- `수정` 버튼 클릭 여부를 마지막 점검 항목으로 체크합니다.

---

## 운영 체크리스트

- [ ] 생년월일: 주민번호 뒷자리까지 연동
- [ ] 우편번호검색 후 주소(동)에 상세 주소 연동
- [ ] 직장(회사): `은행명_예금주명_계좌번호` 형식 준수
- [ ] 직장(회사): `_` 사용 / 공백 없음 / 계좌번호 `-` 포함
- [ ] 전화(직장): 동작구민이면 `동작구` 입력
- [ ] 외국인 여부 확인(뒷자리 시작 `6`)
- [ ] 외국인인 경우 참고사항에 `외국인_6` 형태 추가
- [ ] 외국인 주민번호 입력 규칙 적용
- [ ] 최종 반영 시 **저장이 아닌 수정 버튼** 클릭
